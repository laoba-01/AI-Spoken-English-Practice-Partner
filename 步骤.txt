（适用于：Web / 小程序 / APP 后端 + 接口层，对接大模型 AI，可直接给开发团队用）
核心功能范围
用户发语音 → 转文字
AI 对话陪练（英文）
AI 回复文字 → 转语音

Day 1：需求确认 + 架构搭建 + 核心依赖集成
上午（2-3 小时）
确认产品需求
对话场景：日常 / 商务 / 考试？
是否需要实时对话？
语音格式：mp3 /wav/ 16k 采样率
技术选型确认
前端：小程序 / H5/App
后端：Go/Java/Node.js
AI 服务：
语音识别 ASR：阿里云 / 腾讯云 / OpenAI Whisper
大模型：GPT-4o / Doubao / 通义千问
语音合成 TTS：阿里云 / 微软 / OpenAI TTS
下午（3-4 小时）
创建项目结构
数据库表设计（2 张表即可）
user_conversations（对话会话表）
user_messages（消息内容：用户 / AI，文字 + 语音 URL）
集成核心 SDK/API
ASR 语音识别接口
LLM 大模型调用接口
TTS 语音合成接口
完成环境配置：密钥、超时、限流
晚上（1-2 小时）
单元测试：ASR / LLM / TTS 单独调用成功
输出：Day1 成果 = 三个核心 AI 能力可正常调用
Day 2：核心业务逻辑开发 + 接口联调
上午（3-4 小时）
开发核心对话流程：
用户上传语音 → 后端接收
调用 ASR → 得到用户英文文本
构造 prompt（英文陪练、纠错、友好回复）
调用大模型 → 获取 AI 英文回复
调用 TTS → 生成 AI 回复语音
保存对话记录到 DB
Prompt 示例（直接可用）
plaintext
你是专业的AI英语口语陪练教练，用英文自然对话。
如果用户有语法/发音错误，温柔指出并纠正。
保持简短、口语化、适合练习。
